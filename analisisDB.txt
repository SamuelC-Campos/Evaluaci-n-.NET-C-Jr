Análisis y decisiones de base de datos - PRUEBA PARTE A

Objetivo de la prueba A era diseñar un esquema de base de datos relacional y una consulta para encontrar la categoría del producto más recientemente vendido.

El analisis del modelo fue el siguiente:


El diseño incluye tres tablas principales:

	Categoría: Guarda tipos de productos (CodigoCategoria PK, Nombre). Relación 1:N con Producto.

	Producto: Contiene detalles de productos (CodigoProducto PK, Nombre, CodigoCategoria FK). Relación N:1 con Categoría y 1:N con Venta.

	Venta: Registra las transacciones (CodigoVenta PK, Fecha, CodigoProducto FK). Relación N:1 con Producto.


Decisiones de Diseño:

	Se uso enteros como claves primarias (PK) para IDs y claves foráneas (FK) para conectar tablas, asegurando la integridad de los datos.

	Lod tipos de datos fueron: NVARCHAR(100) para texto y DATE para fechas, buscando simplicidad y compatibilidad.


Consulta SQL para la categoria mas reciente vendido:

	SELECT TOP 1 c.Nombre AS Categoria
	FROM Venta v
	JOIN Producto p ON v.CodigoProducto = p.CodigoProducto
	JOIN Categoria c ON p.CodigoCategoria = c.CodigoCategoria
	ORDER BY v.Fecha DESC;

Explicación de la Consulta:
	La consulta une las tablas Venta, Producto y Categoria. Ordena las ventas por Fecha de forma descendente y usa 
	TOP 1 para obtener el Nombre de la Categoria del producto asociado a la venta más reciente.
